The ARPANET archive is an interactive digital exhibition designed to visualize the formative years of the world’s first wide-area packet-switched network. The goal of this project is not just to list dates on a timeline but to create a temporospatial dashboard that lets viewers explore how a few experimental nodes in California grew into the global infrastructure we now know as the Internet. 
By bridging the gap between high-level network topology and specific historical context, the archive allows users to see not just where the nodes were but what they were. I have integrated metadata including hardware specs (e.g. BBN IMPs), key researchers, and the specific academic missions of each site to contextualize the history of Cold War engineering.

One of the deliberate design choices made during development was the use of a minimalist, abstract grid system for the map viewport rather than a literal geographic map of the United States. To use a literal map, the placement of nodes would need to be geographically precise to avoid a sense of disorienting effect. From images I could find available online, even US maps in geometric shapes tend to be divided by states and highlight the actual state boundaries. On a responsive website where screen sizes change constantly, keeping a node pinned to a specific pixel on a background image of a state is a technical issue that I have not yet found a good solution for. 

For now, I opted for an abstract grid paradigm, and this reinforces the idea of the ARPANET as a logical network rather than just a physical one. The grid suggests a blueprint or a technical schematic. It allows the user to focus on the relationships and the sequence of connections between institutions without being distracted by the literal borders of a modern map. As for the color palette, I shifted from a “Matrix-style” neon on black to a palette of parchment whites, deep navy, and a formal crimson. This creates a more user-friendly look for long-form reading and a tone that is closer to a digital museum.

In order to achieve the dynamic effect of the sidebar, and because CSS cannot remember which map node a user just clicked, I utilized a CSS technique called “radio button hack.” A radio button will stay “checked” after the user clicks on it, until the user clicks a different option. I set the radio buttons to display: none, and then use <label> tags and wire them to the invisible radio buttons. So when the user clicks any one of the map nodes, the browser does the secret click on the invisible radio button.

The second technical challenge was allowing users to filter the map by year to see the network grow. I used the same radio button function and linked the timeline buttons at the bottom of the screen to a second set of hidden switches. I categorized every dot and line on the map by year. Then I wrote CSS rules for every possible scenario. For example, selecting 1970 turns on the lights for the 1969 and 1970 elements while keeping the 1973 and 1975 elements inactive.
